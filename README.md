# 番茄章节发布工具

这是一个本地运行的番茄小说章节发布辅助工具。它用于导入已经写好的 Markdown 章节，生成定时发布计划，并通过独立 Chrome 浏览器进入番茄作者后台提交章节。

## 桌面安装版

桌面安装版支持 Windows 10/11（x64）和 macOS，不需要另外安装 Node.js，也不需要手动启动前后端服务。安装后直接打开“番茄章节发布工具”即可使用，但电脑仍需安装 Chrome。

当前 `0.2.x` 安装包未进行商业代码签名，只适合第一阶段自用：

- Windows 首次运行如果出现 Microsoft Defender SmartScreen，请确认安装包来自本项目发布页，然后点击“更多信息”与“仍要运行”。
- macOS 首次运行如果提示无法验证开发者，请在“系统设置 → 隐私与安全性”中找到拦截提示并选择“仍要打开”。
- 覆盖安装新版本不会删除本地设置、Chrome 登录资料、运行日志或小说发布记录。
- 正式对外销售前必须配置 Windows 代码签名和 Apple Developer ID 签名、公证，不能继续分发未签名安装包。

应用内“设置与数据”页面会显示所有数据保存地址，并提供卸载入口。卸载前会列出待清理的完整路径；应用数据始终清理，小说文件夹中的 `.fanqie-publish.json` 默认删除，也可以取消勾选后保留。

## 运行环境

- Node.js 20 或更高版本
- npm
- 本机已安装 Chrome 浏览器

## 安装依赖

在项目根目录执行：

```bash
npm install
```

如果之前依赖安装不完整，尤其是在 Windows 上遇到 Rollup 可选依赖缺失，可以重新安装一次：

```bash
npm install
```

## 启动项目

项目需要同时启动后端 API 和前端页面。请打开两个终端，路径都切到项目根目录。

第一个终端启动后端：

```bash
npm run dev
```

后端默认地址：

```text
http://127.0.0.1:3456
```

第二个终端启动前端：

```bash
npm run dev:web
```

前端默认地址：

```text
http://127.0.0.1:5173
```

浏览器打开前端地址即可使用。

## 章节文件要求

把一本小说的所有章节放在同一个文件夹中，文件夹名会被识别为书名。

工具会自动尝试识别常见章节文件名，不要求所有人都改成同一种格式。支持的常见格式包括：

```text
第001章 开局.md
第1章 开局.md
001 开局.md
001-开局.md
001_开局.md
001. 开局.md
001.md
```

如果文件名没有章节号，例如：

```text
开局.md
夜雨.md
风起.md
```

工具会按文件名自然排序自动编号为第 1、2、3 章，并在导入结果中提示你确认顺序。

如果你的命名格式比较固定，建议在导入时填写“章节命名格式”，工具会优先按你填写的格式解析。格式必须包含：

- `{章节}`：章节数字
- `{章节名}`：章节标题

示例：

```text
第{章节}章_{章节名}.md
第{章节}章-{章节名}.md
{章节}-{章节名}.md
{章节}_{章节名}.md
{章节}. {章节名}.md
```

例如文件名是：

```text
第001章_这算我弄坏的吗.md
第002章_你管这叫魔猿.md
```

导入时填写：

```text
第{章节}章_{章节名}.md
```

如果不填写“章节命名格式”，工具才会使用智能识别和自动编号。

正文建议使用 Markdown：

```markdown
# 第001章 开局

这里是第一章正文。

这里是第二段正文。
```

工具会自动去掉第一行 `# 标题`，只把正文提交到番茄后台。如果正文第一行不是 Markdown 标题，则会保留原文。

## 使用步骤

1. 启动后端和前端。
2. 打开 `http://127.0.0.1:5173`。
3. 在“导入小说文件夹”中填写本地小说文件夹的完整路径。
4. 点击“导入”。
5. 设置起始章节、结束章节和开始日期。
6. 检查每日发布时间，默认是 `09:30`、`12:00`、`19:00`、`19:05`。
7. 点击“生成计划”。
8. 在发布计划表中检查或手动修改每章的日期和时间。
9. 点击“开始发布”，在最终确认弹窗中确认计划。
10. 工具会打开独立 Chrome 窗口并进入番茄作者后台。
11. 第一次使用时，在工具打开的 Chrome 窗口里手动登录番茄作者后台。
12. 如果工具显示“等待登录”，请先在独立 Chrome 窗口完成登录，再回到工具点击“继续”。
13. 登录完成后，让工具继续操作，不要手动操作这个专用 Chrome 窗口。

## 发布记录

工具会在小说文件夹中生成：

```text
.fanqie-publish.json
```

这个文件用于记录章节状态，例如：

- `pending`：待发布
- `drafted`：已保存草稿
- `scheduled`：已定时发布
- `failed`：发布失败
- `skipped`：已跳过

已经记录为 `scheduled` 的章节，重新生成计划时会自动跳过。

## 注意事项

- 当前工具优先跑通单章定时发布流程，批量发布和暂停继续能力还需要继续完善。
- 番茄后台页面变化可能导致按钮或输入框识别失败。
- 工具不会自动处理验证码、安全验证、登录失效、内容审核失败等情况。
- 章节正文少于 1000 字时，可能无法进入番茄的完整发布流程。
- 发布前建议先用 1 章测试，确认后台显示为“待发布”后，再扩大范围。
- 使用过程中不要操作工具打开的独立 Chrome 窗口，避免流程中断。
